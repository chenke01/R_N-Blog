# 个人博客系统 - 操作文档

## 快速开始

### 1. 本地运行

#### 方法一：直接打开

1. 下载或克隆项目到本地
2. 双击打开 `index.html` 文件
3. 即可在浏览器中查看博客

> ⚠️ 注意：直接打开文件可能无法加载Markdown文件，建议使用方法二。

#### 方法二：使用本地服务器

1. 安装Node.js（如果未安装）

2. 安装http-server：
   ```bash
   npm install -g http-server
   ```

3. 在项目目录运行：
   ```bash
   http-server -p 8080
   ```

4. 打开浏览器访问：`http://localhost:8080`

#### 方法三：使用Python

```bash
# Python 3
python -m http.server 8080

# Python 2
python -m SimpleHTTPServer 8080
```

### 2. 部署到GitHub Pages

#### 步骤一：创建GitHub仓库

1. 登录GitHub
2. 点击右上角 "+" -> "New repository"
3. 填写仓库名称，如：`my-blog`
4. 选择 Public
5. 点击 "Create repository"

#### 步骤二：上传文件

**方法一：使用Git命令**

```bash
# 克隆仓库
git clone https://github.com/你的用户名/my-blog.git
cd my-blog

# 复制项目文件到仓库目录
# ...

# 提交并推送
git add .
git commit -m "Initial commit"
git push origin main
```

**方法二：使用GitHub网页**

1. 进入仓库页面
2. 点击 "Add file" -> "Upload files"
3. 拖拽所有项目文件
4. 点击 "Commit changes"

#### 步骤三：启用GitHub Pages

1. 进入仓库设置：点击 "Settings"
2. 左侧菜单找到 "Pages"
3. Source 选择 "Deploy from a branch"
4. Branch 选择 "main" 和 "/ (root)"
5. 点击 "Save"

等待几分钟后，你的博客将可以通过以下地址访问：
```
https://你的用户名.github.io/my-blog/
```

## 添加文章

### 1. 创建Markdown文件

在 `posts` 文件夹中创建新的 `.md` 文件，例如：`my-new-post.md`

### 2. 编写Frontmatter

文件开头必须包含Frontmatter信息：

```yaml
---
title: 我的新文章
date: 2024-01-20
tags: [技术, 生活]
author: 你的名字
excerpt: 这是文章的摘要，会显示在卡片上。
---
```

#### Frontmatter字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| title | 是 | 文章标题 | `我的第一篇文章` |
| date | 是 | 发布日期 | `2024-01-20` |
| tags | 是 | 标签数组 | `[技术, JavaScript]` |
| author | 否 | 作者名 | `张三` |
| excerpt | 否 | 文章摘要 | `这是摘要...` |
| image | 否 | 封面图片URL | `https://...` |

### 3. 编写正文

使用标准Markdown语法编写文章内容：

```markdown
# 一级标题

## 二级标题

正文内容...

- 列表项1
- 列表项2

**粗体** 和 *斜体*

[链接文字](https://example.com)

![图片描述](image-url.jpg)

```代码块```
```

### 4. 更新manifest.json

打开 `posts/manifest.json` 文件，添加新文章的文件名：

```json
[
    "getting-started.md",
    "javascript-async.md",
    "2024-plan.md",
    "my-new-post.md"
]
```

### 5. 刷新页面

刷新博客页面，新文章将自动显示。

## 使用标签

### 添加标签

在Frontmatter的tags字段中添加标签：

```yaml
tags: [技术, JavaScript, 前端]
```

### 标签命名建议

- 使用中文或英文
- 保持简洁明了
- 建议使用常用标签：
  - `技术` - 技术文章
  - `生活` - 生活随笔
  - `学习` - 学习笔记
  - `教程` - 教程类
  - `思考` - 思考感悟

### 标签筛选

在首页点击标签按钮即可筛选对应标签的文章。

## 主题切换

### 使用内置主题

点击页面右下角的主题切换按钮，可在以下主题间切换：

1. ◐ **暗黑模式** - 默认主题
2. ○ **白色模式** - 浅色护眼
3. ◈ **赛博朋克** - 紫色霓虹风格，深蓝背景

### 添加自定义主题

1. 打开 `js/theme.js` 文件
2. 在 `themes` 对象中添加新主题：

```javascript
themes: {
    // ... 现有主题
    
    custom: {
        name: '我的主题',
        icon: '○',  // 主题图标
        colors: {
            '--bg-primary': '#your-bg-color',
            '--bg-secondary': '#your-secondary-bg',
            '--bg-tertiary': '#your-tertiary-bg',
            '--bg-card': '#your-card-bg',
            '--text-primary': '#your-text-color',
            '--text-secondary': '#your-secondary-text',
            '--text-muted': '#your-muted-text',
            '--accent': '#your-accent-color',
            '--accent-dim': '#your-dim-accent',
            '--border': '#your-border-color',
            '--border-light': '#your-light-border',
            '--shadow': 'rgba(r, g, b, alpha)',
            '--glow': 'rgba(r, g, b, alpha)'
        }
    }
}
```

3. 保存文件，刷新页面即可使用新主题

## 自定义配置

### 修改博客标题

1. 打开 `index.html`
2. 修改以下内容：

```html
<title>你的博客标题</title>

<!-- 英雄区域 -->
<h1>你的博客标题</h1>
<p>你的博客描述</p>
```

### 修改每页显示数量

打开 `js/app.js`，修改配置：

```javascript
config: {
    postsPerPage: 12,  // 修改为你想要的数量
}
```

## 常见问题

### Q: 文章不显示？

**检查清单：**

1. ✅ 文件是否在 `posts` 文件夹中
2. ✅ 文件扩展名是否为 `.md`
3. ✅ 文件名是否添加到 `manifest.json`
4. ✅ Frontmatter格式是否正确
5. ✅ 是否刷新了页面

### Q: 标签不显示？

**检查清单：**

1. ✅ tags字段是否使用数组格式 `[标签1, 标签2]`
2. ✅ 标签名称是否正确
3. ✅ 是否刷新了页面

### Q: 主题切换无效？

**检查清单：**

1. ✅ 浏览器是否支持CSS变量
2. ✅ 是否禁用了JavaScript
3. ✅ 清除浏览器缓存后重试

### Q: 代码不高亮？

**检查清单：**

1. ✅ 代码块是否使用正确的语法
2. ✅ 是否指定了语言名称
3. ✅ 网络连接是否正常（需要加载CDN）

### Q: 部署后样式丢失？

**检查清单：**

1. ✅ 文件路径是否正确
2. ✅ 是否使用了相对路径
3. ✅ GitHub Pages是否部署完成

## 高级用法

在Frontmatter中添加image字段：


